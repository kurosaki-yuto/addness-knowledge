# Addness 全機能ナレッジ

> **Addness** = AI時代の行動管理OS
> 「ググると、わかる。アドれば、できる。」
>
> 最終更新: 2026-02-25

---

## 1. Addnessとは何か

Addnessは「できない」状態をAIの力で解消するプロダクト。

行動できない原因は意志や能力ではなく、**以下の3つが曖昧なこと**にある:

1. ゴールの完了状態が不明確
2. ゴールまでの道筋が整理されていない
3. 詰まったときの戻り方が決まっていない

これを「思考の構造化 + AIサポート」で解決するのがAddness。

---

## 2. 全体構造 ― 5つのページと役割

| ページ | 一言でいうと | いつ使う |
|--------|-------------|---------|
| **ゴール詳細** | ゴールを「今日から動ける状態」に変える | ゴールの定義・分解・基準設定のとき |
| **実行タブ** | 今日やることだけに集中して実行する | 毎日の作業開始〜終了まで |
| **通知** | 自分のゴールに影響する変化に気づく | 他メンバーやAIからのアクションを確認 |
| **メンバー一覧** | チーム全体の状況を俯瞰する | メンバーの稼働・進捗を確認したいとき |
| **設定** | 組織の管理とカスタマイズ | 管理者がプラン・メンバー・連携を管理 |

基本サイクル: **ゴール詳細で思考を整理 → 実行タブで行動 → メンバー一覧/通知で全体管理**

## 3. ゴール詳細ページ

> 漠然とした目標を「今日から動ける状態」に変換する場所

### 3-1. ゴールの状態管理（最重要概念）

Addnessのゴールには2つの状態がある:

| 状態 | 意味 | 実行タブに出る？ |
|------|------|----------------|
| **分解中** | 何をすればいいかまだ曖昧。完了基準やサブゴールを整理している段階 | 出ない |
| **実行可能** | やることが明確で、あとは実行するだけ | 出る |

**「実行可能」にするための条件:**
- 完了の基準（DoD）が明確に言語化されている
- 必要なサブゴールがすべて洗い出されている
- 各サブゴールの責任者が決まっている
- 完了までの道筋がイメージできる

切り替え方法: 画面左上のラベルをクリックでトグル

### 3-2. 完了の基準（DoD: Definition of Done）

「このゴールは何をもって完了とするか」を明文化するエリア。

- カレンダーアイコン横の `>` で展開
- リッチテキストで完了条件を記述（自動保存）
- AIに「完了の基準を更新して」と伝えると自動作成も可能

**良い完了基準の3原則:**

| 原則 | NG例 | OK例 |
|------|------|------|
| 誰が見ても判断できる | 顧客満足度が向上している | NPSスコアが50以上 |
| 検証可能である | ちゃんとテストした | 全テストケースが合格 |
| 複数条件はリスト化 | 採用が完了 | 内定承諾書受領済 / 入社日確定 / PC手配完了 |

### 3-3. タイトル設定

タイトルをクリック → 編集モード → Enter で保存。

**良いタイトルの3原則:**
1. **具体的**: `売上アップ` → `Q1で売上1億円を達成する`
2. **期限/数値入り**: `採用活動` → `6月末までにエンジニア3名を採用する`
3. **行動が想像できる**: `顧客満足度の件` → `NPSを40→60に改善する`

NG: 「〜の件」「〜について」「〜関連」

### 3-4. 期日設定

- 左上の `P --` をクリック → カレンダーモーダル（YYYY/MM/DD）
- 期日が近いゴールは実行タブで優先表示される
- 期日超過ゴールは赤く表示される

### 3-5. KPI設定

- 三点メニュー `...` → 「KPIを設定する」
- 指標名（例: 新規顧客数） + 目標値（例: 100社） + 現在値を定期更新
- **先行指標**（商談数、提案数）をKPIにするとアクションに直結しやすい
- 1ゴールにつき1〜3個が適切

### 3-6. 成果物アップロード

- 三点メニュー `...` → 「成果物をアップロード」
- ファイルアップロード or リンク登録
- **AIがコンテキストとして活用する**ので積極的に登録すべき

### 3-7. サブゴール

- ツリー構造で無制限にネスト可能
- 「+ タイトルを「〜する」の形で入力」で追加
- 各サブゴールに個別のアサイン、期日、完了チェック、コメントが付く

### 3-8. コメントスレッド

- 画面下部のコメント欄に入力 → `Cmd+Enter` で送信
- `@` でメンバーをメンション → 通知が届く
- ファイル添付（`+` ボタン）、音声入力（マイクアイコン）対応
- **ゴールに紐づく議論・決定事項を蓄積する場所**（Slackで流れない）

### 3-9. 権限設定

| 権限 | 人数 | できること |
|------|------|-----------|
| **責任者** | 1名 | 全操作可能。進捗の説明責任を負う |
| **編集者** | 複数 | ゴール配下の情報を編集可能 |
| **メンバー** | 複数 | 閲覧のみ |

右上のユーザーアイコン群 → メンバー追加/権限変更

### 3-10. アクティビティログ

ゴールに関する全操作の履歴（作成/編集/完了/期日変更/メンバー変更/コメント/状態変更/成果物アップロード）

### 3-11. 関連するゴール

「+ 追加」で別のゴールとリンクを設定できる。

### 3-12. 三点メニュー `...` の全機能

**親ゴール:**

| 機能 | 説明 |
|------|------|
| 完了 | ゴールを完了状態にする |
| Objective付け替え | 上位ゴールとの紐付けを変更 |
| 期日を設定 | 期日モーダルを開く |
| 成果物をアップロード | ファイル/リンクを登録 |
| KPIを設定する | KPI設定モーダルを開く |
| 繰り返し | 日次/週次/月次の定期ゴール化 |
| URLをコピー | ゴールの共有リンクをコピー |
| ゴールの一括編集 | ツリー俯瞰表示で一括操作 |
| AIエージェントをアサイン | AIが自律的にゴールに取り組む（後述） |
| アーカイブ | 非表示化（削除ではない） |
| 削除したゴール復元 | 削除済みゴールのリストア |
| 削除 | 完全削除（赤字表示） |

**サブゴール:**
完了 / アサイン / Objective付け替え / 期日設定 / 繰り返し / URLをコピー / ゴールの一括編集 / アーカイブ / 削除

## 4. 実行タブページ

> 「今日やるべきこと」だけに集中する。認知負荷を最小化する場所

### 4-1. 画面構成

| エリア | 内容 |
|--------|------|
| **左パネル** | 今やるべきゴール + 今日のToDo（ツリー表示） |
| **右パネル** | AIチャット（実行中のゴールについてサポート） |

### 4-2. ゴール vs ToDo（重要な違い）

| | ゴール | ToDo |
|---|---|---|
| **起点** | トップダウン（ビジョン→部門→チーム→個人） | ボトムアップ（現場で思いついたこと） |
| **Why** | 上位ゴールとの紐付けで明確 | 曖昧な場合もある |
| **構造** | ツリーの一部 | 独立して存在 |
| **例** | 「Q1売上目標達成」の子として「提案資料作成」 | 「電話する」「会議室予約」「本を読む」 |

ToDoとして登録したものも、重要なら後からゴールに紐づけるのが理想。

### 4-3. 主要機能

- **▶ 再生ボタン**: 各タスクを即時実行開始 → AIチャットと自動連携
- **タイマー機能**: スタート/ストップ/完了で作業時間を計測
- **定常ToDo**: 毎日自動生成される繰り返しタスク（歯車 → 設定で管理）
- **Googleカレンダー連携**: その日のカレンダーイベントをゴールとして自動取込
- **未完了タスク自動引き継ぎ**: やり残しを翌日に自動持ち越し

### 4-4. 設定（歯車アイコン）

- Googleカレンダーイベントをゴールとして出力（ON/OFF）
- カレンダーイベントを追加する組織（ドロップダウン選択）
- 定常ToDoの追加・編集・無効化

## 8. AI機能（Addy）

### 8-1. AIチャット ― コア機能

**どこから使える？**
- ゴール詳細ページ: 「AIと相談」ボタン → 右パネルに表示
- 実行タブ: 右パネルに常時表示

**AIモデル（4段階）:**

| モデル | 特徴 |
|--------|------|
| **Addy2.0 nano** | 軽量・高速（Beta） |
| **Addy2.0 light** | バランス型（Beta） |
| **Addy2.0** | 標準モデル |
| **Addy2.0 Max** | 最高性能（Beta / 有料制限あり） |

**AIにできること:**
- サブゴールの作成・提案
- 完了基準（DoD）の作成・更新
- KPIの作成・更新
- コメントの作成・更新
- 成果物の登録
- 壁打ち・レビュー・思考整理

**入力方法:**
- テキスト入力
- 音声入力（マイクアイコン）
- ファイル添付（`+` ボタン）
- `/` でゴールをメンション、`@` でメンバーをメンション

**チャット履歴:** AIパネル左上のハンバーガーメニューで過去の全会話一覧

### 8-2. AIエージェントアサイン

- ゴールの三点メニュー `...` → 「AIエージェントをアサイン」
- **AIが自律的にゴールに取り組む**
- 進捗はチャットで確認可能
- 完了時に通知が届く
- AIがDoD観点でギャップ分析してコメントを自動投稿する

### 8-3. Skills一覧

- AIパネル右上のスパークルアイコンから開く
- カスタムプロンプトテンプレートを作成・管理
- CoT（Chain of Thought）タグ対応
- 検索・新規作成・編集・削除が可能
- 繰り返し使うプロンプトをスキル化して再利用

### 8-4. アクション履歴

- AIパネル右上のフィルターアイコンから開く
- AIが実行したアクションのログを確認

### 8-5. AIとの対話 ― 4つのコツ

| # | コツ | ポイント |
|---|------|---------|
| 1 | **コンテキストをしっかり伝える** | 背景・制約・現状を詳しく。音声入力が楽でおすすめ |
| 2 | **自分の言葉で回答する** | 選択肢をポチるだけで終わらない。理由を添える |
| 3 | **スレッドは適度に区切る** | 長くなると精度が落ちる。AIの「新スレッド提案」に従う |
| 4 | **AIの提案を尊重する** | 「基準を整理しよう」と言われたら焦らず従う。結果的に早い |

## 11. 基本原則 ― 行動できないときの対処法

行動できない = Addnessが上手く使えていないだけ。対処はシンプル:

```
1. ゴールの「完了の基準」を見直す
2. サブゴールを追加・整理する
3. サブゴールが「実行可能」になるまで 1 と 2 を繰り返す
4. それでも解消しなければ、上位ゴールの完了基準を見直す
```

**この手順で、全てのゴールは必ず達成可能。**
